# Output format (group conventions)

## Weekly regime report (suggested layout)

- Timestamp (KST)
- Regime + score (0–100)
- Cash floor/range
- Allowed actions this week
- Key drivers:
  - Index breadth summary
  - BTC trend/vol summary
  - Top 3 headlines + categories + severity

## KRW formatting

If reporting KRW values to the user, **truncate decimals** (e.g., `12,345.67원 -> 12,345원`).

## PnL

If you are reporting account status (e.g., from an exchange), include **unrealized PnL** when possible and label it explicitly as unrealized.
